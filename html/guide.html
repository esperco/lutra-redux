Esper Living Style Guide

This is the Esper (CSS) style guide. It serves as playground to mess around with CSS and as an example of "best practices" as we currently understand them. Inspect this using the Chrome inspector to see what's going on. We're not very strict about styling at the moment though. That might change if we ever get a full-time designer.

We currently target IE10+. That means you can use stuff like Flexbox.

As far as general principles go, less is more. It's better to overload semantic selectors like <input> and <button> than use an abundance of classes.

Dropdowns

Requires React component

Dropdowns are actually a special case of a modal for us. To make dropdowns responsive on mobile, only one <Dropdown> component at a time is visible. On desktop, dropdowns will position themselves up, down, left, right, or center automatically based on relative screen positioning.
